# Ozon Orders Finder

Tampermonkey userscript для фильтрации заказов на странице архива Ozon по диапазону цены, с поддержкой сканирования всех годов и локальной базой данных заказов.

## Установка

1. Установите расширение [Tampermonkey](https://www.tampermonkey.net/) в браузер
2. Создайте новый скрипт и вставьте содержимое файла `ozon-orders-finder.user.js`
   или откройте в браузере `https://github.com/p166/ozon-orders-finder/raw/refs/heads/main/ozon-orders-finder.user.js` и подтвердите установку
3. Откройте страницу архива заказов: `https://www.ozon.ru/my/orderlist?selectedTab=archive`

## Разрешения

Скрипт не запрашивает специальных разрешений (`@grant none`). Работает только на странице `https://www.ozon.ru/my/orderlist?selectedTab=archive*` и использует:
- `localStorage` для хранения настроек, базы данных заказов и статусов сканирования
- `fetch` для запроса деталей заказов
- `MutationObserver` для отслеживания динамической подгрузки заказов

## Возможности

### Фильтр по цене

- Введите минимальную и/или максимальную цену в рублях
- Нажмите **Найти** — подходящие заказы подсветятся зеленой рамкой, первый найденный заказ автоматически прокрутится в видимую область
- При желании включите **Скрыть не подходящие** — карточки заказов вне диапазона будут скрыты
- Нажмите **Сбросить**, чтобы убрать выделение и показать все заказы
- Настройки фильтра (диапазон цен, флаг скрытия) сохраняются в `localStorage` и автоматически применяются при следующем открытии страницы
- Фильтр автоматически применяется к динамически подгружаемым заказам при скролле

### Сканирование заказов

- **Сканировать** — собирает детали всех заказов выбранного года (названия товаров, цены, даты, суммы) и сохраняет в локальную базу данных
- **Сканировать все года** — последовательно сканирует все доступные годы архива, автоматически переключаясь между годами
- Во время сканирования показываются цветные индикаторы статуса для каждого года:
  - Серый — не сканирован
  - Оранжевый — запрашиваются карточки товаров
  - Синий — запрашиваются детали заказов
  - Зеленый — сканирование завершено
  - Красный — ошибка
- Кнопка **Стоп** позволяет прервать сканирование в любой момент
- Сканирование можно возобновить после перезагрузки страницы — очередь и статусы сохраняются в `localStorage`
- При пересканировании данные за год полностью обновляются

### Все заказы (модальное окно)

- Кнопка **Все заказы** открывает модальное окно с объединенной таблицей всех сканированных заказов за все годы
- Поддерживается поиск по названию товара в реальном времени
- Сортировка по любому столбцу (год, дата, название, цена, номер заказа) по возрастанию/убыванию
- При клике на строку открывается детальная страница заказа на ozon.ru
- Кнопка **Очистить** удаляет все сканированные данные и статусы

## Хранилище данных

Скрипт использует следующие ключи `localStorage`:

- `ozon_orders_finder_settings` — настройки фильтра (мин/макс цена, флаг скрытия, активен ли фильтр)
- `ozon_orders_finder_data` — база данных заказов по годам (`{ "2024": { orders: [...], scannedAt: "..." } }`)
- `ozon_orders_finder_scan_queue` — очередь годов для сканирования при режиме "Сканировать все года"
- `ozon_orders_finder_scan_status` — статусы сканирования для каждого года
- `ozon_orders_finder_total_processed` — счетчик обработанных заказов при пакетном сканировании (временный)

## Архитектура

- Сканирование одного года: последовательная пакетная обработка по 10 заказов с задержкой 800ms между пакетами, поддержка `AbortController` для отмены
- Сканирование всех годов: навигация по годам через `window.location.href` с параметром `selectedYear`, обработка каждой страницы через `MutationObserver` и `processScanQueue`
- Отслеживание смены года в интерфейсе через `MutationObserver` на блоке фильтров
- Автоматическое применение фильтра к новым блокам заказов через `MutationObserver` на `document.body`
